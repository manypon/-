# Сборка в .exe (PyInstaller)

Соберётся и будет работать, но с оговорками ниже. Код уже поправлен под
это (см. `_ensure_dependencies` — при запуске из .exe автоустановка
библиотек через pip отключается сама, так как в собранном виде это не
нужно и не сработает).

## 1. Установка PyInstaller

```
pip install pyinstaller
```

## 2. Команда сборки

Собирай через `--onedir`, а не `--onefile` — так надёжнее для
customtkinter и playwright (их доп. файлы иногда криво распаковываются
из one-file архива во временную папку при каждом запуске):

```
pyinstaller --noconfirm --onedir --windowed --name "YandexMapsScanner" ^
  --collect-all customtkinter ^
  --collect-all playwright ^
  main.py
```

(`^` — перенос строки в cmd.exe; если собираешь в PowerShell, замени на
` \` в конце строки или пиши одной строкой).

Результат появится в `dist/YandexMapsScanner/YandexMapsScanner.exe`.
Раздавать/переносить нужно **всю папку** `dist/YandexMapsScanner`, а не
только .exe — там рядом лежат все нужные .dll и ресурсы.

## 3. Важные нюансы

- **`--collect-all customtkinter` обязателен** — у customtkinter есть
  json-файлы тем (цвета, шрифты), которые PyInstaller сам не подхватывает.
  Без этого флага при запуске .exe будет ошибка вида
  `FileNotFoundError: ...customtkinter/assets/themes/...`.
- **`--collect-all playwright` обязателен** — у playwright есть
  собственный небинарный driver (Node.js-процесс внутри пакета), который
  тоже не подтягивается автоматически.
- **Браузер не нужно бандлить** — мы используем уже установленный на
  компьютере Яндекс.Браузер/Chrome/Edge через `executable_path`, а не
  встроенный браузер Playwright. Поэтому `playwright install` не нужен ни
  тебе, ни конечному пользователю — экономит сотни МБ в сборке.
- **Первая сборка — тестируй с консолью**, без `--windowed`, чтобы видеть
  ошибки в реальном времени, если что-то не соберётся с первого раза.
  Уберёшь `--windowed`, когда всё стабильно заработает.
- **Размер**: из-за pandas/openpyxl готовая папка весит примерно
  150–250 МБ — это нормально, не баг.
- **Антивирусы**: PyInstaller-сборки (особенно `--onefile`) иногда ловят
  ложные срабатывания эвристики Windows Defender/антивирусов. `--onedir`
  реже вызывает такие проблемы, чем `--onefile`.

## 4. Быстрая проверка перед раздачей

Запусти `YandexMapsScanner.exe` на чистой машине (или хотя бы не из-под
VSCode/терминала с активным venv) — так сразу видно, не забыл ли
PyInstaller что-то важное.
